# Product Context — HMA

## Why This Product Exists
<!-- TODO: Confirm with user -->
HMA was originally a Blogger-based site with a cyberpunk/neon design. It's being completely rebuilt as a modern, professional Next.js web application. The platform serves a community (likely Myanmar-based, given Burmese language in docs) that needs access to shared Apple ID accounts for software downloads.

## Problems It Solves
1. **Apple ID Access** — Provides free Apple IDs for users who may not have their own (e.g., for downloading region-locked apps)
2. **Credential Management** — Centralized place to manage and distribute Apple ID credentials securely
3. **Communication** — Blog for sharing iOS tips, tutorials, and announcements with the community

## Target Audience
<!-- TODO: Confirm with user -->
- Myanmar tech community members who need Apple ID access
- Users connected via Telegram channel and Facebook group
- People interested in iOS tips and tutorials

## User Experience Goals
- **Clean & Professional**: Moved away from cyberpunk/neon to Apple-inspired minimalism
- **Trust**: Professional feel appropriate for an account-sharing platform
- **Mobile-First**: Responsive design that works on all devices
- **Fast**: Quick access to credentials with copy-to-clipboard functionality

## User Journeys

### New User
1. Admin creates account for user
2. User receives credentials (via Telegram/Facebook?)
3. User logs in → sees homepage with latest Apple IDs and blog posts
4. User browses Apple IDs, copies credentials they need

### Admin
1. Logs in → sees admin dashboard with stats
2. Manages Apple IDs (add/edit/delete/toggle status)
3. Creates user accounts
4. Writes blog posts
5. Reviews issue reports from users
6. Manages site settings (alerts, social links)
